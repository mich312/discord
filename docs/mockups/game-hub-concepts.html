quorum — game hub concepts
quorum · design exploration nº2

The circle becomes a game hub.

Same encrypted bones — chat, calls, invite-only circles — but the center of gravity moves to playing together: a shelf of games that live on other servers, launchable into the room with your call and chat docked beside them. Two visual directions below, each with full screens. Scroll for the concept notes and my recommendation at the end.

DIRECTION 01

Afterdark — the private arcade

OLED-black with a blue cast, coral signal color, presence-green reserved for “live right now”. Cover art carries the energy; chrome stays quiet and glassy. The mono voice of old quorum survives — but demoted to captions, stats and the security whisper-lines, where it belongs.

The hub replaces “home base”. Tonight’s plan up top with a live countdown; the shelf shows every game the circle registered — web games launch embedded, native servers (Minecraft, Factorio) show live status pulled by your client, never the relay. Green means someone’s in it right now.
Launching pulls the room with you. The game (here a web chess game hosted on its own server) takes the stage; call controls and the room’s chat dock right. The host chip is honest: quorum seals your chat and voice — the game server sees its own traffic.
Chat warms up. Round avatars, grouped messages, reactions — and the payoff feature: a game invite is a first-class message. One tap and you’re on the board with the call still running.
DIRECTION 02

The Den — game night at someone’s place

The other honest answer to “private circle that plays together”: not an arcade, a den. Warm lamplight palette, brass accent, serif display like a well-worn club card. Same layout bones as Afterdark — proof the structure works in more than one skin.

Same hub, warmer room. Everything sits a little softer — bigger radii, lamplight gradients, serif voice. Good fit if the circles are more board-game-night than ranked-ladder.
Chat in the Den. Identical information design to Afterdark’s chat — only the material changes. Pick the mood, keep the bones.
how it would actually work

Concept notes — games that live on other servers

two kinds of game

Activities vs. servers

Web games (chess, cards, .io arcade) embed as a sandboxed iframe with a tiny postMessage contract — identity, roster, who’s host — the same architecture as Discord’s Embedded App SDK and Matrix widgets. Native servers (Minecraft, Factorio) get a status card instead: your client pings the server directly for player count and latency, and “Join” deep-links or copies the address.

launch from the call

Voice comes first

Discord’s game store died; launching games from inside the call you’re already in is the model that stuck (+1M game-launch days/month after they surfaced it better). The lounge call is the ambient session — games start into it, survive it, and drop a “bob started X — Join” card into chat for late arrivals.

sealed metadata

The E2EE angle stays honest

The game registry (names, URLs, cover colors) travels inside MLS like channel names — the relay never learns what your circle plays. What changes: connecting to a game host reveals your IP and traffic to that host. The UI says so on the card and in the game view, in the same honest voice the app already uses.

presence

Every status line is a button

“playing Hex Gambit · 41 min” isn’t decoration — it ends in Join/Watch. For a 6-person circle, presence-as-launch-affordance (Steam’s friends rail) is the single highest-leverage pattern: the hub answers “who’s around and what are they doing” before you ask.

migration

Additive, not big-bang

Discord’s March 2025 relayout taught the lesson: users punish everything-at-once reskins. The hub ships as a new tab replacing “home base”; chat keeps its muscle memory and gets restyled tokens only. Both directions here share one layout skeleton — the skin is swappable.

joining

Jackbox-grade entry

Any running game is joinable in one tap from chat, the shelf, or a presence line. Lobbies show each joiner’s avatar pop in; the host holds the only Start button. No installs, no forms — the circle is already authenticated.

My take

Lead with Afterdark. It keeps quorum’s discipline (quiet chrome, mono captions for the security whisper-lines, restraint) but finally gives the product warmth and a pulse — glow strictly meaning “live”, cover art carrying the energy, coral instead of caution-tape yellow. It reads as a place to be, not a terminal to operate.

The Den is the dark horse — if your circles are more board-game-night than ranked-ladder, its lamplight/brass mood is more distinctive than any arcade look; nobody else ships that. It’s also cheap to keep: both directions are one token set apart, so it could simply be the second theme (replacing “paper”).

Build order I’d suggest: ① restyle tokens + chat (lowest risk, biggest felt change), ② game hub page with server-status cards (no new protocol work — the client pings hosts directly), ③ embedded activities with the postMessage contract, ④ invite cards + rich presence. Say the word and I’ll start on ①+② as a real branch.
